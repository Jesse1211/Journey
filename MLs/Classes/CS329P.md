### 1. Supervision
#### Semi-Supervised Learning SSL
- Train with a small set of labeled data & a large set of unlabeled data
	- Reason: “Once you know how labels behave on some parts of the manifold, you can propagate that information smoothly along the whole manifold using unlabeled data.”
- Assumption: 
	- similar input should have similar output (labels / clusters)
	- manifold - Data lies near a low-dimensional manifold & The label/function changes smoothly along that manifold.
- $$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{sup}} + \lambda \,\mathcal{L}_{\text{unsup}} = \frac{1}{|L|} \sum_{(x_i, y_i) \in L} \ell\big(f_\theta(x_i), y_i\big) + \lambda \frac{1}{|U|} \sum_{x_j \in U} R\big(f_\theta, x_j\big)$$
	- $R$ regularization term - Consistency regularization - similar inputs shouldn't change much prediction 
    	- $$f_\theta(x) \approx f_\theta(\tilde{x})$$
Example - Self-training
- let model generate labels for unlabeled data, then retrain the model using its own guesses.
1. Setup data pools 
   - $$L = \{(x_i, y_i)\}, \quad U = \{x_j\}$$
2. Train initial model on the small labeled set L using normal supervised learning 
   - $$\mathcal{L}_{\text{sup}} = \frac{1}{|L|} \sum_{(x_i, y_i) \in L} \ell(f_\theta(x_i), y_i)$$
3. Generate pseudo-labels for unlabeled data $x_j \in U$
	- $$c(x) = \max_k p(y = k \mid x) = \max_k pf_{\theta}(x)$$
		- $c(x) \geq \lambda_{\text{high}}$: high confidence -> keep predictions  
		- $c(x) \leq \lambda_{\text{low}}$: low confidence -> active learning
		- $\lambda_{\text{low}} \lt c(x) \lt \lambda_{\text{high}}$: medium confidence -> handle separately
	1. create pseudo-label if confident, pseudo-labels = predicted label 
      - $$\hat{y}(x) = \arg\max_k p(y = k \mid x)$$
	2. Active learning: query low-confidence (hard / uncertain points)
		- handle because they will change my model the most by human
		1. For each unlabeled $x \in U$, compute uncertainty $u(x)$ (margin/entropy.)
		2. Rank unlabeled points by $u(x)$ (most uncertain first).
		3. Pick the top B examples subject to your human labeling budget.
		4. Ask a human (oracle) to label these → you get true labels y.
		5. Add to labeled set **A** and remove from U.
		- Margin - small margin = more uncertain 
    		- $$\text{margin}(x) = p_{(1)}(x) - p_{(2)}(x)$$
			- p1 = largest probability, p2 = second largest
		- Entropy - large entropy = more uncertain 
    		- $$H(x) = - \sum_k p(y = k \mid x) \log p(y = k \mid x)$$
	3. Retrain / fine-tune with weighed loss 
         - $$\mathcal{L} = \mathcal{L}_{\text{true}} \;+\; \lambda_{\text{pseudo}} \,\mathcal{L}_{\text{pseudo}} =  \frac{1}{|L| + |A|} \sum_{(x, y) \in L \cup A} \ell(f_\theta(x), y)+ \lambda_{\text{pseudo}} \frac{1}{|P|} \sum_{(x, \hat{y}) \in P} \ell(f_\theta(x), \hat{y})$$
		- Original labeled: L
		- Actively labeled (true human labels): A
		- Pseudo-labeled (model guesses): P
		- $\lambda \in [0, 1]$ “how much we trust” the pseudo-labels
4. Handle “medium confidence” unlabeled points 
	- option1: keep unlabeled - for future training
	- option2: consistency regularization
		1. Create an augmented / perturbed version $~\tilde{x}$ of the same sample
		2. impose a consistency loss 
            - $$\mathcal{L}_{\text{consistency}}(x) = \| f_\theta(x) - f_\theta(\tilde{x}) \|^2$$
		3. Add to overall loss (with weight) 
            - $$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{true}} + \lambda_{\text{pseudo}} \mathcal{L}_{\text{pseudo}} + \lambda_{\text{cons}} \mathbb{E}_{x \in U_{\text{mid}}} \mathcal{L}_{\text{consistency}}(x)$$
#### Weak Supervision ==我觉得可以做运维==
training on lots of inexpensive, noisy, approximate labels (from rules, existing models, logs, crowds, etc.) instead of relying only on a small set of perfectly clean human annotations, often combining multiple weak sources and modeling their noise.
- Examples of weak sources
	- Examples of weak sources:
		- Heuristics / rules: “If email contains FREE MONEY → label spam”
		- Existing models: Use an old classifier or off-the-shelf API as a teacher.
		- Distant supervision: If a sentence mentions “Barack Obama” and “USA”, label as “born in USA”.
- labels:
	- Noisy: sometimes wrong.
	- Inexact: label applies to a broader unit than the real target.
	- Incomplete: some positive examples are unlabeled (false negatives).

Simple noise model - merge sources to **probabilistic labels** 
$$\hat{P}(y \mid \Lambda(x)) = [0.8, 0.15, 0.05]$$
- $$\Lambda(x) = [\lambda_1(x), \lambda_2(x), \dots, \lambda_m(x)]$$
- $\lambda$: labeling functions => returns noisy label in $\{1, 2, ..., K\}$
- $\Lambda$: weak labels for x (votes from labeling functions) => rule 1 says 80%, rule 2 says 15%, ...
- $\epsilon$: noise rate
- option1: Directly train on $(x_i, \tilde{y}_i)$ and hope the model is robust to noise (with regularization, large scale, etc.)
- option2: 1. Model the noise and try to infer latent true labels $y_i$​ from multiple noisy sources.
### 2. Data Transformation - Normalization
#### Min-max normalization
Rescale values to a fixed range $[0, 1]$ or $[−1,1]$. Good for distance-based methods, or models that assume bounded inputs. 
$$x' = \frac{x - \min(x)}{\max(x) - \min(x)}$$
Pros:
- focus absolute scale & use distances
- KNN, K-means, distance-based methods.
Cons:
- Sensitive to outliers: extreme outlier makes the max huge & everything else close to 0.
- new test data too big / small makes normalized values can be <0 or >1
#### Z-score normalization
Make the feature have mean 0 and standard deviation 1 when Features have very different scales.
$$z = \frac{x - \mu}{\sigma}$$
- x feature
- $\mu$ mean
- $\sigma$ std
Pros:
- common with gradient-based models or models with regularization (L1/L2), where scale matters a lot.
    - Linear models (logistic regression, linear regression),
    - Neural networks,
    - PCA (principal component analysis),
    - Anything that assumes something like “Gaussian-ish” behavior.
- Less sensitive to outliers _than_ min–max (but still affected).
Cons:
- No fixed range; values can be from $-\infty$ to $+\infty$ 
- Affected by outliers
#### Log scaling
Compress large values, expand small positive values, reduce skewness (for heavily skewed distributions), used before applying Z-score or other normalizations (unfixed ranges) 
$$x' = \log(x + c)$$
Pros:
- Reduces right skew (lots of small values, few huge ones).
- stabilize variance.
- Often improves linear models on heavy-tailed distributions.

Cons:
- Requires $x < -c$, usually people require $x \geq 0$ and add c if there are zeros

#### Reduce words 
Stemming
- chop off word endings using simple rules, without caring much about grammar or “real” words. => _“playing”_ -> `play`
- For rough normalization
- Good for search engines, quick-and-dirty text retrieval, simple models.

lemmatization
- reduce words to dictionary base form (lemma) using vocabulary + morphology + part-of-speech (POS) => (“I saw”) -> lemma see
- When caring semantic meaning and correctness.
- used pre-deep-learning era; nowadays big language models often skip explicit lemmatization and work directly on tokens or subwords.

Tokenization
- breaking raw text into tokens (basic units) that a model/process can work with. 
	- `Raw text → [tokens] → features for your model`
- Basic (word-level) tokenization
	- split on whitespace and punctuation.
	- `["I", "love", "playing", "football", ",", "but", "I", "hate", "running", "."]`
- Subword / BPE tokenization (modern deep learning)
	- BPE (Byte Pair Encoding), WordPiece
	- Handles rare / new words by decomposing them. 
		- `"unbelievable" => ["un", "believ", "able"]`
	- Keeps vocabulary size manageable (e.g. 30k–50k tokens) instead of millions of words.
- Character-level tokenization
	- each character is a token: `"hello" → ["h", "e", "l", "l", "o"]`
